Client-side for validation

  • Never trust data passed to your server from a client. Even if validation is working Clients can still alter network traffic.
  • HTML5 form controls come with basic properties like "required" that make sure a neccessary field is not left empty."Pattern" uses regular expressions to define which characters are allowed or not, and in what order.
  • Javascript can also be used for form validation using the Constraint Validation API
  • Client side validation is more about the user experience than making 100% sure data is valid and not malicious Server side validationg is the main defense againt bad data.

Using Fetch

  • The fetch API has replace an older technology known as XMLHttpRequest
  • Fetch can be used for many thing other than just fetching data from a server. It can also be used to upload JSON data or other files, or edit data the alread exists.
  • It is important to be aware of fetch headers, as some may be neccessary when implementing some technologies.
