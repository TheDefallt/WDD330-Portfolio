Notes

  • Place primary controls like menus within the easy to reach areas of the one handed grip.Move menus to the bottom to increase reachability.
  • The hamburger menu is not as readily recognized as we think. The text Menu with a boarder around it gives it a button appearance and is more recognizable to the user.
  • Make for mobile first reduces cognative load and makes things easier to use. Mobile forces us to focus on what is important.
  • Add placeholders in the input fields instead of labels to the side. Making them float labels is even better because the user remebers what information they are typing.
  • Larger checkboxes are better than small ones.
  • Only request user input that is needed. If they only want phone contanct, only ask for their phone number not email or street address.
  • Pagination is better than one long page.
  • Make forms have steps with a progress bar if a paginated for is needed. Autocomplete on your forms is a good idea as well. Keeps user from abandoning form and eliminates errors.
  • Temporarily show passwords. Prevents support calls. Show passwords by default, but give option to hide.
  • If possible see if progressing through the form with only one input box will work. Space apart long numbers automatically like phone or credit card numbers.
  • Present user with best keyboard for data input. Present number keyboard when only numbers are needed.
  • If you plan on using ads, don't use pop ups everyone hates them. Don't autoplay videos or audio either.
  • Don't cram too many ads on a page. Bar adds between text is better.
  • Make them get larger when clicked. Videos work better than stills. Ad can show up behind text and be viewed through a small window.
