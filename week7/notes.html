Chapter 11

  • Functions can have methods and properties of thier own.
  • Using the call function allows you to pass objects as the first argument. If more parameters are required, they should be designated after the first object paramenter. The object must always be first.
  • Custom properties are added using dot notation for example, a function with the name addNumbers can have a property named description added in this way, addNumbers.description = 'A string describing the function'.
  • Memoization helps cache results of expensive computations so they can just be used again later without needing to compute them again.
  • When using code from others, it may be best to use the "use strict" command on only your code inside an IIFE function
  • Promises can be a better way of managing asynchronous code than using multiple callbacks
  • The three rules of pure functions are: 1. Return values of the functions are only base on the arguments provided. 2. There are no side effects. Functions only make none destructive changes to values or other data. It only makes non-destructive data transformations 3. Referential transparancy. Given the same arguments, a pure function will always return the same result.
