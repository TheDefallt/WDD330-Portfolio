Chapter 9

  • All JavaScript environments have a global object. All global variables and functions are members of this object. In browser environments this object is the window object.
  • Just like there is a DOM, there is also a BOM (Browser Object Model) that gives developers access to browser and computer screen information.
  • windows.navigator.userAgent gives data on the browser that is being used
  • windows.location can be used to get or set the web location of the browser. This can be use to redirect users to different pages.
  • windows.location has many functions to get specific information about the location such as protocol, host, hostname, and port.
  • Windows can be controlled with methods like close(), moveTo(), and resizeTo().
  • windows.screen will return information about the screen like height and width. This can be very useful for mobile webpages.
  • Be aware of internation laws like the EU Cookie Directive that requires permission from the user before cookies are used
  • To use a cookie, assign it to the JavaScript cookie jar with document.cookie = ''. Using just document.cookie will read all current cookies

Chapter 14

  • data- attribute is a way of embedding data in a web page. They are private to pages.
  • HTML5 contains API's to help developers access hardware information like Web Storage, Geolocation, speed, altitude, etc.
  • Web workers allow processes to run in the background instead of just in a single thread which is JavaScript's default.
  • Service workers allow a worker script to run in the background, but is also able to intercept network requests.
